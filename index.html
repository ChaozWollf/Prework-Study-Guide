HTML

  • The head element contains informantion about the webpage.
  • The body element represents the visible content shown to the user.
  • creates structure of the webpage
  • when possible use semantic elements
  • HTML has nested elements with in other elements
  • the nested Elements are called child elements
  • the elements that hold all the children elements are called parent elements

CSS

  • CSS is Cascading Style Sheets
  • Declarations contain 2 important parts
  • Important parts of a declaration are CSS property, and value added to that property
  • Margin indicates how much space is outside of element
  • padding indicates how much space we want around the content inside the element.
  • a selector defines the element or attributes to which a rule will apply
  • example: div{color:blue;text align;justify}.....div is the selector
  • when using external CSS make sure to have a command
  • * = wildcard symbol
  • * applies to ALL elements
  • can combine two selectors seperated by a coma
  • link rel defines the relationship between the current document and the linked document

Git

  • git is a version control system
  • git is used in git hub to maintain a workflow
  • git status: check what branch we are on
  • git checkout -b branch-name: creates a new branch and switches to it
  • feature branch holds clode for a single feature
  • every git repo starts with a default branch named "main"
  • developer Branch can handle multiple feature branches that haven't been incorperated in the main branch yet
  • Requiring that branches be reviewed and approved assures that the new features won't break the main code base.
  • github flow 1. create a branch
  • 2. make changes
  • 3. create a pull request
  • 4. address review comments
  • 5. merge your pull request
  • 6. delete your branch
